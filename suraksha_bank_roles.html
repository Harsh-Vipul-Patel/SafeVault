Role-Based Page Access Layout

This document defines which pages, features, and Oracle PL/SQL stored procedures are accessible to each role. Access is enforced at three layers: the Application Tier (session token + RBAC middleware), the Oracle DB Tier (stored procedure caller validation via SYS_CONTEXT), and the UI Tier (hidden/locked navigation items).

Callable Oracle PL/SQL Procedures
⚡
sp_internal_transfer(p_sender, p_receiver, p_amount, p_user)
Executes atomic fund transfer with SERIALIZABLE isolation + SELECT FOR UPDATE. Validates sender ownership via SYS_CONTEXT before proceeding.
CUSTOMERTELLER
⬇️
sp_initiate_external_transfer(p_account, p_amount, p_ifsc, p_acc_no, p_mode)
Phase 1 of two-phase external transfer. Debits account, creates PENDING record. REGEXP_LIKE validates IFSC format before any debit occurs.
CUSTOMERTELLER
📊
sp_generate_statement(p_account_id, p_from_date, p_to_date)
Generates paginated transaction history for a date range. Account ownership is verified — customer cannot pass another customer's account_id.
CUSTOMERTELLERMANAGER
🔑
sp_change_password(p_user_id, p_old_hash, p_new_hash)
Validates old password hash, updates USER_SESSIONS to revoke all active sessions, sets new bcrypt hash. Atomic — session invalidation and password change in one TX.
CUSTOMERTELLERMANAGER
Denied Actions & Reasons
🚫
Cannot View Other Accounts
Stored procedures reject any account_id not linked to the caller's customer_id. Returns ORA-20001, not empty result.
🚫
Cannot Open New Account
Account creation requires a Teller with branch_id context and KYC verification. Not available in self-service portal.
🚫
Cannot Access Audit Logs
AUDIT_LOG and PROCEDURE_EXECUTION_LOG are restricted to Manager/Admin roles. SELECT privilege not granted to customer DB role.
🚫
Cannot Deposit/Withdraw Cash
Physical cash operations (sp_deposit, sp_withdraw) require a Teller's employee session with verified branch_id. Cannot be initiated online.

Data Scope: Branch-Scoped (branch_id embedded in session token)

A Teller's branch_id is set at login via DBMS_SESSION.SET_CONTEXT('SURAKSHA_CTX','branch_id','BRN-MUM-003'). All stored procedures read this context value and reject operations for accounts or customers outside the teller's assigned branch.

Callable Oracle PL/SQL Procedures
💰
sp_deposit(p_account_id, p_amount, p_teller_id)
Credits account balance, inserts TRANSACTION record (type=CREDIT), fires audit trigger. Validates teller's branch_id matches account's home branch or teller has cross-branch override.
TELLERMANAGER
💸
sp_withdraw(p_account_id, p_amount, p_teller_id)
SELECT FOR UPDATE WAIT 5 acquires row lock, re-validates balance atomically (TOCTOU prevention), debits account, inserts TRANSACTION record. Raises ORA-20002 if balance insufficient after lock.
TELLERMANAGER
🆕
sp_open_account(p_customer_id, p_type_id, p_initial_deposit, p_teller_id)
Creates new ACCOUNTS record, inserts initial deposit TRANSACTION, links to home_branch_id from teller's session context. Generates formatted account_number via sequence.
TELLERMANAGER
⬆️
sp_submit_dual_approval(p_operation_type, p_payload_clob, p_requested_by)
INSERTs high-value operation into DUAL_APPROVAL_QUEUE with status=PENDING. The originating teller cannot approve their own submission — enforced by CHECK constraint on approver_id != requested_by.

Exclusive Oracle PL/SQL Procedures
✅
sp_approve_dual_queue(p_queue_id, p_manager_id, p_note)
Validates approver's role = BRANCH_MANAGER via SYS_CONTEXT. CHECK constraint ensures approver != original requester. Executes the queued operation atomically then marks queue record APPROVED.
MANAGER ONLY
🔒
sp_set_account_status(p_account_id, p_new_status, p_manager_id, p_reason)
Updates ACCOUNTS.status to FROZEN/DORMANT/CLOSED. trg_audit_status_change fires automatically, capturing old/new status, actor, and timestamp in AUDIT_LOG. Closure rejected if balance > 0.
MANAGER ONLY
🌐
sp_approve_external_transfer(p_transfer_id, p_manager_id)
Phase 2 of two-phase external transfer. Updates PENDING_EXTERNAL_TRANSFERS status to SETTLED, generates simulated UTR reference via Oracle sequence, updates linked TRANSACTION to COMPLETED.
MANAGER ONLY
↩️
sp_reject_external_transfer(p_transfer_id, p_manager_id, p_reason)
Rejects transfer, reverses the escrowed debit by crediting the source account, inserts CREDIT reversal TRANSACTION, updates transfer status to REJECTED. Full atomic rollback simulation.
MANAGER ONLY

Role · SYSTEM_ADMIN
System Administrator
System-wide configuration, user management, and infrastructure oversight. Critical separation of duties: System Admins CANNOT initiate or approve financial transactions — they manage the system that runs transactions, never the money itself.
16
Pages Accessible
6
Pages Denied
6
DB Procedures
Data Scope: System-Wide (Configuration Only — Zero Financial Transaction Access)

System Admin has Oracle DB privileges for DDL changes (via controlled scripts), SELECT on all tables for monitoring, and EXECUTE on configuration/maintenance procedures. The Oracle DB role SURAKSHA_ADMIN_ROLE explicitly excludes EXECUTE on any financial stored procedure (sp_deposit, sp_withdraw, sp_internal_transfer, etc.).
